Exercice 2 : Propriétés d’une mémoire (framebuffer) — mode présentation interactive

Énoncé — Framebuffer (mémoire tampon d’image)

Image 1280 × 1024, couleur RGB avec 8 bits par couleur (R, G, B). On veut : (1) taille mémoire minimale, (2) temps minimal d’envoi à 100 Mbit/s.

Astuce : change les paramètres pour “voir” l’impact

Idée clé (très exam)

  • Nombre de pixels = largeur × hauteur
  • Bits par pixel (bpp) = canaux × bits par canal
  • Taille (octets) = pixels × (bpp / 8)
  • Temps d’envoi = (taille en bits) / (débit en bit/s)

Pour RGB 8 bits/canal → bpp = 3 × 8 = 24 bpp = 3 octets/pixel.

Illustration : un pixel RGB (cliquable)

Clique sur R/G/B pour mettre en évidence → tu comprends pourquoi c’est 3 octets/pixel.

Question 1 — Taille minimale du framebuffer (octets / Mo)

On calcule pixels puis octets/pixel → taille totale.

Réponse attendue (cas de l’énoncé)

1280 × 1024 = 1 310 720 pixels
RGB 8 bits/canal → 3 octets/pixel
Taille = 1 310 720 × 3 = 3 932 160 octets ≈ 3.75 Mo (MiB)

Schéma : comment on “remplit” la mémoire

La mémoire est “linéaire”, même si l’image est 2D → c’est le contrôleur vidéo qui reconstruit la trame.

Question 2 — Temps minimal d’envoi sur un réseau à 100 Mbit/s

Débit = (nombre de bits) / temps ⇒ temps = bits / débit.

Animation + calcul automatique

Paramètres réseau

Taille (bits) (= octets × 8)
—
Débit (bit/s)
—
Temps minimal (= bits / débit)
—

Réponse attendue (cas de l’énoncé)

Taille = 3 932 160 octets → 31 457 280 bits
Débit = 100 × 10⁶ bit/s
Temps ≈ 0.3146 s ≈ 0.3 s

Ici on suppose un envoi “idéal” : sans overhead (protocoles), ni pertes, ni latence supplémentaire.

Animation : “tuyau” réseau (lance / stop)

La barre bleue = progression de l’envoi. Le “tuyau vert” = flux instantané.

Synthèse — Réponses de l’Exercice 2

Les formules à mémoriser + résultats numériques du cas donné.

Fiche révision

Formules

  • Npx = L × H
  • bpp = canaux × bits/canal
  • octets = Npx × (bpp/8)
  • temps = (octets×8) / débit

Résultat Q1 (énoncé)

Pixels : 1280×1024 = 1 310 720
Octets/pixel : RGB 8b → 3
Taille = 3 932 160 octets ≈ 3.75 Mo

Résultat Q2 (énoncé)

Bits : 3 932 160 × 8 = 31 457 280
Débit : 100 Mbit/s = 100×10⁶ bit/s
Temps ≈ 0.3146 s ≈ 0.3 s

À dire clairement en copie (phrase type)

“L’image contient L×H pixels. Un pixel RGB codé sur 8 bits par couleur occupe 24 bits = 3 octets. La taille minimale du framebuffer vaut donc Npx×3 octets. Le temps minimal d’envoi sur un réseau de débit D vaut (taille en bits)/D.”
