Exercice 1 : Systèmes à microprocesseur (mode présentation interactive)

1) Éléments principaux d’un système à microprocesseur

À retenir : un système à µp = CPU + horloge + mémoires + bus + interfaces E/S.

Astuce : clique sur un élément pour le mettre en évidence

Réponse (liste structurée)

  • Microprocesseur (CPU) : exécute les instructions et traite les données.
  • Horloge : synchronise toutes les opérations (rythme du système).
  • Mémoires : stockent programmes et données (ROM/Flash, RAM, cache…).
  • Bus : transportent adresses, données et signaux de contrôle.
  • Interfaces Entrées/Sorties (E/S) : relient clavier, afficheur, capteurs, réseau, etc.

Schéma global (interactif)

Le BUS regroupe généralement : bus d’adresses, bus de données, bus de contrôle.

2) Rôle de l’Unité de Commande (UC) et de l’Unité de Traitement (UT)

UC = “chef d’orchestre”, UT = “calcul et manipulation des données”.

Clique : UC vs UT

Réponse attendue

  • Unité de commande (UC) : séquence le déroulement des instructions (orchestration).
  • Unité de traitement (UT) : traite les données (opérations arithmétiques/logiques, registres de travail).

Mémo : UC “séquencer”, UT “traitement des données”.

Schéma interne CPU (interactif)

Mémo : UC = contrôle ; UT = calcul.

3) Rôle des registres / UAL / pile — et leur unité (UC ou UT)

Clique sur une ligne pour afficher un rappel rapide.

Tableau + aide interactive

Tableau de synthèse

Composant Rôle Unité
Compteur de programme (PC) Stocke l’adresse de la prochaine instruction à exécuter UC
Registre d’instruction (IR) Stocke temporairement l’instruction machine en cours de traitement UC
Registre d’état (Flags) Indique l’état du processeur / résultat des opérations (flags) UT
Accumulateur Stocke opérandes et résultats des opérations arithmétiques et logiques UT
Pile (Stack) Stocke des données critiques (adresse de retour CALL/RET, etc.) UC
UAL (ALU) Exécute les opérations arithmétiques et logiques UT

Conseil : PC & IR = “pilotage”, ALU/accumulateur/flags = “calcul”.

Mini-illustration : “où ça se place ?”

Rappel express (utile en examen)

  • PC : “où est la prochaine instruction ?” → UC
  • IR : “quelle instruction je traite ?” → UC
  • Pile : “où revenir après CALL ?” → UC
  • UAL + Accu + Flags : “je calcule et je mémorise le résultat” → UT

4) Différences clés : CISC vs RISC • Harvard vs Von Neumann • Classique vs Pipeline

Compare “philosophie d’instructions”, “organisation mémoire”, et “mode d’exécution”.

Comparaisons + pipeline
CISC instructions complexes
  • Instructions souvent longues et puissantes (une instruction = plusieurs micro-étapes).
  • Objectif historique : réduire le nombre d’instructions (au prix de complexité matérielle).
RISC instructions simples
  • Instructions simples, souvent de durée plus régulière.
  • Objectif : vitesse matérielle et exécution efficace (souvent pipeline-friendly).

Harvard vs Von Neumann (schéma interactif)

Classique vs Pipeline (animation + “cycle”)

Classique : une instruction à la fois.
Pipeline : plusieurs instructions avancent en parallèle (Fetch/Decode/Execute).

Lecture “cycle par cycle” : à chaque cycle, une instruction avance d’une étape.

5) Les 3 phases principales d’exécution d’une instruction

Fetch → Decode → Execute (et on recommence pour l’instruction suivante).

Clique sur une phase

Réponse attendue

  1. Recherche (Fetch) de l’instruction en mémoire
  2. Décodage (Decode) par l’unité de commande
  3. Exécution (Execute) de l’opération

Mini-méthode pour expliquer aux étudiants

  • Fetch : “je lis la consigne”
  • Decode : “je comprends ce que je dois faire”
  • Execute : “je fais l’action (calcul, lecture/écriture, saut…)”

Animation + explication (interactif)

Ces 3 étapes : fetch, decode, execute.
