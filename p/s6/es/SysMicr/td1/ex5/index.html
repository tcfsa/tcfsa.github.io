Exercice 5 — Décodage d’adresse (A23..A0, D7..D0)

RAM 8 Mbit • ROM 8 Mbit • P1=8 Ko • P2=4 Ko • P3=2 Ko (CS bas/haut)

Énoncé (résumé) + stratégie

Bus d’adresse 24 bits → espace total 2²⁴ = 16 Mo. Bus de données 8 bits (1 octet par adresse).

On découpe en blocs (zones) de 1 Mo

Données

  • RAM : 8 Mbit → 1 Mo
  • ROM : 8 Mbit → 1 Mo
  • P1 : 8 Ko
  • P2 : 4 Ko
  • P3 : 2 Ko
Ici on suppose l’usage classique en architecture mémoire : 1 Ko = 2¹⁰ octets, 1 Mo = 2²⁰ octets, et 8 Mbit = 2²³ bits = 2²⁰ octets = 1 Mo.

Les broches /CS_RAM, /CS_ROM, /CS_P1 sont actives à 0 (barre), tandis que CS_P2 et CS_P3 sont actives à 1.

Plan de placement (simple & standard)

  • RAM dans les adresses les plus basses : bloc 0x0xxxxx
  • Puis ROM : bloc 0x1xxxxx
  • Puis P1 : bloc 0x2xxxxx
  • Puis P2 : bloc 0x3xxxxx
  • Puis P3 : bloc 0x4xxxxx

Décodage très simple : on utilise uniquement A23..A20 (16 blocs de 1 Mo). Les périphériques “occupent” un bloc de 1 Mo (→ redondance, vue plus loin).

Q1 — Taille du bus d’adresse de chaque composant

Nombre de lignes d’adresse internes = log2(nombre d’octets adressables).

Résultat : RAM/ROM=20, P1=13, P2=12, P3=11

Calcul

Composant Taille Octets Lignes d’adresse
RAM 8 Mbit 1 Mo = 2²⁰ A0..A19 (20 bits)
ROM 8 Mbit 1 Mo = 2²⁰ A0..A19 (20 bits)
P1 8 Ko 2¹³ A0..A12 (13 bits)
P2 4 Ko 2¹² A0..A11 (12 bits)
P3 2 Ko 2¹¹ A0..A10 (11 bits)

Bus données = 8 bits ⇒ 1 adresse = 1 octet.

Schéma (adresse 24 bits)

Q2 — Bits utilisables pour le décodage + nombre de zones

Les bits non utilisés par l’adressage interne des gros blocs (1 Mo) servent au décodage.

Décodage : A23..A20 → 16 zones de 1 Mo

Réponse

  • RAM et ROM font 1 Mo = 2²⁰ → elles utilisent A0..A19.
  • Il reste donc 4 bits : A20, A21, A22, A23.
  • Nombre de combinaisons (zones) : 2⁴ = 16.
  • Chaque zone = 1 Mo (car offset interne 20 bits).

Zones : 0x0xxxxx à 0xFxxxxx (16 Mo au total)

Visualisation : 16 zones de 1 Mo

Q3 — Associations “en utilisant moins de zones possibles”

On choisit 5 zones parmi 16 : RAM, ROM, P1, P2, P3.

0→RAM, 1→ROM, 2→P1, 3→P2, 4→P3

Choix de mapping (simple)

Zone (A23..A20) Adresses (hex) Composant
0x0 0x000000 → 0x0FFFFF RAM (1 Mo, occupe tout le bloc)
0x1 0x100000 → 0x1FFFFF ROM (1 Mo, occupe tout le bloc)
0x2 0x200000 → 0x2FFFFF P1 (8 Ko, miroir dans le bloc)
0x3 0x300000 → 0x3FFFFF P2 (4 Ko, miroir dans le bloc)
0x4 0x400000 → 0x4FFFFF P3 (2 Ko, miroir dans le bloc)

C’est volontairement “simple” : on ne raffine pas le décodage avec A19..Ax → donc redondance pour P1/P2/P3.

Carte mémoire (interactive)

Q4 — Fonctions de décodage (CS)

On décode A23..A20. Attention : /CS actifs bas, CS actifs haut.

Équations “propres” + forme De Morgan

Table de sélection (par zone)

Zone A23..A20 Signal actif
RAM0000/CS_RAM = 0
ROM0001/CS_ROM = 0
P10010/CS_P1 = 0
P20011CS_P2 = 1
P30100CS_P3 = 1

Pour les zones non utilisées, aucun composant ne doit être sélectionné.

Équations (avec A23..A20)

Notation : Z3Z2Z1Z0 = A23A22A21A20

Pour un signal actif bas qui doit valoir 0 pour une combinaison donnée, on peut écrire directement une “NAND logique” : /CS = ¬( condition_en_1 ).
Pour un signal actif haut, on écrit : CS = condition_en_1.

/CS_RAM = ¬(¬A23 · ¬A22 · ¬A21 · ¬A20)

/CS_ROM = ¬(¬A23 · ¬A22 · ¬A21 · A20)

/CS_P1 = ¬(¬A23 · ¬A22 · A21 · ¬A20)

CS_P2 = (¬A23 · ¬A22 · A21 · A20)

CS_P3 = (¬A23 · A22 · ¬A21 · ¬A20)

Ces formes correspondent à un décodage “1 parmi 16” sur A23..A20 (ex : décodeur 4→16).

Q5 & Q6 — Espace occupé + représentation mémoire (adresses remarquables)

On distingue “taille réelle” vs “espace réservé par le décodage (zone)”.

Les périphériques sont “miroités” dans 1 Mo

Tailles réelles vs espace réservé

Composant Taille réelle Base Fin réelle Espace réservé (zone)
RAM 1 Mo 0x000000 0x0FFFFF 1 Mo (tout le bloc)
ROM 1 Mo 0x100000 0x1FFFFF 1 Mo (tout le bloc)
P1 8 Ko 0x200000 0x201FFF 1 Mo (miroirs)
P2 4 Ko 0x300000 0x300FFF 1 Mo (miroirs)
P3 2 Ko 0x400000 0x4007FF 1 Mo (miroirs)

Les fins réelles se calculent : fin = base + taille - 1.

Adresses remarquables (exemples)

  • Début RAM : 0x000000 — Fin RAM : 0x0FFFFF
  • Début ROM : 0x100000 — Fin ROM : 0x1FFFFF
  • Début P1 : 0x200000 — Fin réelle : 0x201FFF
  • Début P2 : 0x300000 — Fin réelle : 0x300FFF
  • Début P3 : 0x400000 — Fin réelle : 0x4007FF
Dans les zones P1/P2/P3, les adresses au-delà de la fin réelle retombent sur la même mémoire interne (redondance / mirroring).

Q7 — Redondance (mirroring) des éléments

Redondance = (espace réservé) / (taille réelle) pour P1/P2/P3.

P1=128× • P2=256× • P3=512×

Calcul

  • Espace réservé par zone = 1 Mo = 1024 Ko
  • P1 = 8 Ko → redondance = 1024/8 = 128
  • P2 = 4 Ko → redondance = 1024/4 = 256
  • P3 = 2 Ko → redondance = 1024/2 = 512
RAM et ROM : pas de redondance ici, car leur taille (1 Mo) remplit exactement leur zone.

Intuition

Si P1 n’utilise que A0..A12, alors les bits A13..A19 n’entrent pas dans son adressage interne → changer ces bits ne change pas le registre interne sélectionné.

Q9 — Schéma de câblage (principe)

Connexion des bus + décodeur 4→16 sur A23..A20 + adaptation actifs bas/haut.

Schéma fonctionnel clair

Connexions essentielles

  • Bus données D7..D0 vers RAM, ROM, P1, P2, P3 (en parallèle).
  • Bus adresses :
    • RAM/ROM : A0..A19
    • P1 : A0..A12
    • P2 : A0..A11
    • P3 : A0..A10
  • Décodage : A23..A20 → sélection 1 parmi 16 zones.
  • Signaux CS :
    • Sorties actives bas → directement pour /CS_RAM, /CS_ROM, /CS_P1.
    • Pour CS_P2, CS_P3 (actifs haut) → inversion (NOT).

Schéma (SVG) — “décodeur + bus”

En pratique, un composant type 74HC154 (4→16, sorties actives bas) correspond exactement à ce besoin.
